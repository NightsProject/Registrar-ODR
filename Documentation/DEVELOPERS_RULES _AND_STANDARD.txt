## 3. CODING STANDARDS

### Python Standards (Flask Backend)

#### Naming Conventions
- **Files**: snake_case.py
- **Classes**: PascalCase
- **Functions/Variables**: snake_case
- **Constants**: UPPER_SNAKE_CASE
- **Blueprint names**: descriptive_snake_case
- **Route names**: descriptive_endpoint_names

#### Code Organization
```python
# Required imports order
# 1. Standard library
import os
import json
from datetime import datetime

# 2. Third-party packages
import psycopg2
from flask import Blueprint, request, jsonify
from flask_jwt_extended import jwt_required, get_jwt_identity

# 3. Local imports
from app.services.logging_service import Logger
from app.utils.decorator import admin_required

# Blueprint definition
example_bp = Blueprint('example', __name__, url_prefix='/api/example')

# Route handlers
@example_bp.route('/endpoint', methods=['GET'])
@jwt_required()
@admin_required
def handle_endpoint():
    """Function description."""
    try:
        # Implementation
        pass
    except Exception as e:
        # Error handling
        pass
```

#### Flask-Specific Standards
- **Blueprints**: One blueprint per feature module
- **Error Handling**: Use custom error handlers in utils/error_handlers.py
- **Database**: Always use connection pooling (g.db_conn)
- **Authentication**: Use @jwt_required() decorator for protected routes
- **Input Validation**: Validate all input data
- **Response Format**: Return JSON responses with consistent structure

#### Database Standards
```python
# Database connection pattern
def get_db_connection():
    return g.db_conn

# Query execution pattern
def execute_query(query, params=None):
    conn = get_db_connection()
    cursor = conn.cursor()
    try:
        cursor.execute(query, params)
        if query.strip().upper().startswith('SELECT'):
            return cursor.fetchall()
        else:
            conn.commit()
            return cursor.rowcount
    finally:
        cursor.close()

# Database operations should be in try-except blocks
```

## 4. API STANDARDS

### Backend API Design

#### Response Format
```json
{
    "success": true,
    "data": {
        // Response data
    },
    "message": "Operation successful",
    "timestamp": "2024-01-01T00:00:00Z"
}
```

#### Error Response Format
```json
{
    "success": false,
    "error": {
        "code": "ERROR_CODE",
        "message": "User-friendly error message",
        "details": "Technical details for debugging"
    },
    "timestamp": "2024-01-01T00:00:00Z"
}
```

#### HTTP Status Codes
- **200**: Success
- **201**: Created successfully
- **400**: Bad request (validation errors)
- **401**: Unauthorized (authentication required)
- **403**: Forbidden (insufficient permissions)
- **404**: Not found
- **409**: Conflict (duplicate data)
- **422**: Unprocessable entity (business logic error)
- **500**: Internal server error

### Frontend API Integration

#### API Service Pattern
```javascript
// services/api.js
import axios from 'axios';

const api = axios.create({
    baseURL: process.env.REACT_APP_API_URL || 'http://127.0.0.1:8000',
    withCredentials: true,
    headers: {
        'Content-Type': 'application/json',
    },
});

// Request interceptor
api.interceptors.request.use(
    (config) => {
        // Add auth token if available
        return config;
    },
    (error) => {
        return Promise.reject(error);
    }
);

// Response interceptor
api.interceptors.response.use(
    (response) => response.data,
    (error) => {
        // Handle common errors
        if (error.response?.status === 401) {
            // Redirect to login
            window.location.href = '/login';
        }
        return Promise.reject(error);
    }
);

export default api;
```

## 5. DATABASE STANDARDS

### PostgreSQL Standards

#### Table Naming
- **Tables**: snake_case (lowercase with underscores)
- **Columns**: snake_case
- **Primary Keys**: id (SERIAL PRIMARY KEY)
- **Foreign Keys**: {table_name}_id
- **Timestamps**: created_at, updated_at (TIMESTAMP DEFAULT NOW())

#### Query Standards
```sql
-- Always specify columns, avoid SELECT *
SELECT id, name, email, created_at 
FROM users 
WHERE status = 'active' 
ORDER BY created_at DESC 
LIMIT 50;

-- Use proper indexing
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status_created ON users(status, created_at);

-- Use transactions for multiple operations
BEGIN;
    INSERT INTO requests (user_id, document_type) VALUES (1, 'transcript');
    UPDATE user_stats SET request_count = request_count + 1 WHERE user_id = 1;
COMMIT;
```

#### Connection Management
- Use connection pooling (max 10 connections)
- Always close cursors after use
- Use context managers when possible
- Handle connection failures gracefully

## 6. SECURITY STANDARDS

### Authentication & Authorization
- **JWT Tokens**: Use secure, httpOnly cookies
- **Token Expiry**: 1 hour for access tokens
- **Password Security**: Never store plaintext passwords
- **Session Management**: Use Flask-Session with secure settings
- **CSRF Protection**: Enabled for all state-changing operations
- **Role-Based Access**: Implement role checks for admin routes

### Input Validation
```python
# Backend validation example
from marshmallow import Schema, fields, validate

class UserRegistrationSchema(Schema):
    email = fields.Email(required=True)
    password = fields.Str(required=True, validate=validate.Length(min=8))
    first_name = fields.Str(required=True, validate=validate.Length(max=50))
    last_name = fields.Str(required=True, validate=validate.Length(max=50))
```

```javascript
// Frontend validation example
const validateEmail = (email) => {
    const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return re.test(email);
};

const validatePassword = (password) => {
    return password.length >= 8;
};
```

### Data Sanitization
- **SQL Injection**: Use parameterized queries only
- **XSS Prevention**: Escape all user input in templates
- **File Uploads**: Validate file types and sizes
- **API Responses**: Sanitize sensitive data

## 10. PERFORMANCE STANDARDS

### Backend Performance
- **Database Queries**: Use indexes, optimize queries
- **Connection Pooling**: Proper pool size configuration
- **Caching**: Implement caching for frequently accessed data
- **Response Time**: API responses < 200ms for simple queries

### Frontend Performance
- **Bundle Size**: Keep initial bundle < 2MB
- **Lazy Loading**: Use React.lazy for code splitting
- **Image Optimization**: Optimize and compress images
- **API Calls**: Debounce search inputs, paginate large lists

## 11. ERROR HANDLING STANDARDS

### Backend Error Handling
```python
from app.utils.error_handlers import APIError

@app.errorhandler(404)
def not_found(error):
    return jsonify({
        'success': False,
        'error': {
            'code': 'NOT_FOUND',
            'message': 'Resource not found'
        }
    }), 404

@app.errorhandler(500)
def internal_error(error):
    # Log error for monitoring
    app.logger.error(f'Server Error: {str(error)}')
    return jsonify({
        'success': False,
        'error': {
            'code': 'INTERNAL_ERROR',
            'message': 'An unexpected error occurred'
        }
    }), 500
```

### Frontend Error Handling
```javascript
// Custom hook for error handling
const useErrorHandler = () => {
    const [error, setError] = useState(null);

    const handleError = (error) => {
        console.error('Error:', error);
        setError(error.message || 'An error occurred');
    };

    const clearError = () => setError(null);

    return { error, handleError, clearError };
};

// Usage in components
const Component = () => {
    const { error, handleError } = useErrorHandler();

    const handleApiCall = async () => {
        try {
            await api.post('/endpoint');
        } catch (err) {
            handleError(err);
        }
    };

    return (
        <div>
            {error && <ErrorAlert message={error} />}
            {/* Component content */}
        </div>
    );
};
```
